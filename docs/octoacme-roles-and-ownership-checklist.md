# OctoAcme — Roles & Ownership Checklist

Use this checklist at **project kickoff** to confirm that every required role is identified, named, and understands its responsibilities before planning begins. An incomplete roster is one of the most common sources of delivery delays.

---

## Step 1 — Identify Required Roles

For each role below, mark whether it is needed for this project and, if so, record the name or team responsible.

| Role | Required? (Y/N) | Person / Team | Notes |
|---|---|---|---|
| Project Manager (PM) | | | |
| Product Manager (PdM) | | | |
| Developer(s) | | | |
| QA Lead / Test Engineer | | | |
| Scrum Master / Agile Coach | | | |
| UX Designer / Researcher | | | |
| DevOps / Platform Engineer | | | |
| Engineering Manager / Tech Lead | | | |
| Security / Compliance | | | |
| Data / Analytics | | | |
| Executive Sponsor / Stakeholder | | | |

> **Tip:** Not every project needs every role. Mark a role "N" only if you have explicitly confirmed it is not applicable.

---

## Step 2 — Confirm Responsibilities Are Understood

For each required role, confirm the following with the person/team filling it:

- [ ] They have read the [Roles & Personas doc](octoacme-roles-and-personas.md) entry for their role
- [ ] They understand their key responsibilities for **this** project
- [ ] They know who to escalate to and who to coordinate with
- [ ] They have access to the project board, repo, and communication channels
- [ ] They are aware of their expected availability and sprint commitment

---

## Step 3 — Complete the RACI Matrix

After confirming roles, fill in the [RACI Template](octoacme-raci-template.md) to assign Responsible, Accountable, Consulted, and Informed designations for each major activity.

- [ ] RACI matrix drafted for core project activities
- [ ] RACI reviewed and agreed upon by all role holders
- [ ] RACI linked or embedded in the Project One-pager

---

## Step 4 — Kickoff Sign-off

- [ ] All required roles confirmed and named
- [ ] Any gaps (e.g., role needed but person TBD) logged as risks in the Risk Register
- [ ] Roles and RACI shared with stakeholders
- [ ] This checklist saved in the project's `docs/` folder or linked from the Project Charter
